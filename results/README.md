# results

Preliminary data from the Qwen2.5-0.5B-Instruct inference experiment in Section 5.2 of the paper. Generated by `analysis/collect_preliminary.py`.

## Files

- `preliminary_0p5b.jsonl` — N=30 trials, one JSON record per line.
- `failure_modes.png` — bar chart of successes vs failure modes.
- `summary.txt` — text version of the same breakdown.

## Numbers

30 trials, 0 successes. Failure modes:

```
illegal_chars       15   50.0%
wrong_value         13   43.3%
no_answer_line       1    3.3%
numbers_not_in_set   1    3.3%
```

"illegal_chars" and "numbers_not_in_set" both correspond to the paper's failure mode (b) ("Answer line uses integers outside the provided set"). `illegal_chars` specifically covers the "= 42" appended-equality pattern the paper cites.

## A note on ordering vs the paper

The paper (Section 5.2) describes three failure modes in the order a > b > c (no parseable Answer line > outside-set > wrong value). This N=30 batch ordered them b > c > a. The difference is that the demo's prompt template explicitly instructs "End your response with a single line 'Answer:'...", which suppresses mode (a) relative to the informal ordering the paper gives. The general claim the paper makes — three distinct failure modes, success rate near zero — holds here.

## Reproducing

```
cd TinyTinyZero
source demo/.venv/bin/activate
python analysis/collect_preliminary.py --n 30
python analysis/summarize.py
python analysis/categorize.py
```

Takes ~2 minutes on an Apple Silicon laptop with the model cached.
